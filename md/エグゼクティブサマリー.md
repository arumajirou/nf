# AI駆動型開発プロジェクト管理 - エグゼクティブサマリー

## 概要

このドキュメント群は、**人間の気分や稼働状況に依存せず、毎日確実にPDCAサイクルを回す**AI駆動型開発チームの構築方法を提供します。

### 調査結果

**Claude Code（AnthropicのCodex製品）について:**

Claude Codeは、ターミナルで動作するエージェント型コーディングツールで、以下の特徴があります:

1. **コードベース全体の理解** - プロジェクト構造を把握し、文脈を理解
2. **直接的な実行能力** - ファイル編集、コマンド実行、Git操作を直接実行
3. **自然言語での指示** - 技術的な詳細を自然な言葉で指示可能
4. **並列実行サポート** - Git worktreesを使用した複数インスタンス同時実行
5. **推奨4段階ワークフロー** - リサーチ→プラン→実装→仕上げ

**公式ベストプラクティス（Anthropic）:**
- 段階的なワークフローが品質向上の鍵
- 複数エージェントの協働が効果的
- テスト駆動開発との相性が良い
- 明確な完了基準と検証ステップが重要

## 問題と解決策

### 現状の問題

```
人間の管理者 ────→ 現状把握 ────→ 指示出し
     ↓                              ↓
  気分に依存                    ChatGPT-5
     ↓                              ↓
  進捗が滞る                    Claude Code
```

**ボトルネック:**
- 人間が動かないと進まない
- 非エンジニアが技術判断を求められる
- 日次PDCAサイクルが不安定

### 提案する解決策

```
┌─────────────────────────────────┐
│   自動化レイヤー（専用GPTs）      │
│   - 日次レポート自動生成          │
│   - タスクキュー自動管理          │
│   - 異常時のみ人間にエスカレート   │
└─────────────────────────────────┘
         ↓ 自動    ↑ 例外時のみ
┌─────────────────────────────────┐
│   ChatGPT-5 Thinking (PM)       │
│   - 技術的判断                   │
│   - タスク分解                   │
└─────────────────────────────────┘
         ↓ 指示    ↑ 結果
┌─────────────────────────────────┐
│   複数Claude Code インスタンス    │
│   並列実行で効率化               │
└─────────────────────────────────┘
```

**3層の意思決定階層:**

| 層 | 内容 | 人間の関与 |
|----|------|-----------|
| 層1: 完全自動 | lint修正、テスト追加、ドキュメント更新 | 不要（事後報告のみ） |
| 層2: 事前承認 | 新機能追加、小規模アーキテクチャ変更 | 週次/月次レビュー |
| 層3: 即時判断 | ビジネス要件、大規模変更、セキュリティ | 即座にエスカレート |

## 3つの専用GPTsの役割

### GPT A: プロジェクトオーケストレーター
**役割:** 人間とAI開発チームの橋渡し

**主要機能:**
- 日次進捗レポート自動生成（非技術者向け）
- 優先度判断とタスクキュー管理
- エスカレーション判定（人間判断が必要な時のみ通知）
- 技術用語 → ビジネス用語の翻訳

**あなたへの価値:**
- **時間節約:** 日次2-4時間 → 15-30分に削減
- **ストレス軽減:** 技術詳細を理解する必要なし
- **透明性:** いつでも進捗を把握可能

### GPT B: Claude Code マネージャー
**役割:** Claude Codeへの最適な指示生成

**主要機能:**
- 抽象的なタスク → 具体的なプロンプトに変換
- Anthropic公式ベストプラクティスの適用
- 複数インスタンスの最適配置
- エラー分析と対策立案

**技術的価値:**
- Claude Codeの能力を最大限に引き出す
- 品質の高い実装を一貫して生成
- 失敗から学習し、プロンプトを継続的に改善

### GPT C: 品質保証エージェント
**役割:** 自動品質チェックとマージ判定

**主要機能:**
- 多層品質評価（コード品質、セキュリティ、パフォーマンス）
- 自動コードレビュー
- マージ可否の自動判定
- 品質トレンド分析

**品質保証:**
- プロダクションレベルの品質を自動保証
- セキュリティリスクの早期発見
- 技術的負債の継続的監視

## 自動PDCAサイクル

### 毎日の自動実行

| 時刻 | アクション | 担当 | 人間の関与 |
|-----|----------|------|-----------|
| 00:00 | 前日サマリー生成 | GPT A | なし（翌朝確認） |
| 01:00 | 次日計画策定 | ChatGPT-5 | なし（自動） |
| 09:00 | 朝報配信 | GPT A | 確認のみ |
| 10:00 | Claude Code実行開始 | Claude Code × 複数 | なし（自動） |
| 18:00 | 夕方進捗報告 | GPT A | 確認のみ |
| 随時 | 異常検知・エスカレーション | 全システム | 必要時のみ |

### 人間が関与するのは

✅ **毎日（5-10分）:**
- 朝報の確認
- 判断が必要な事項への回答（ある場合のみ）
- 夕方報告の確認

✅ **週次（30分-1時間）:**
- 週次サマリーのレビュー
- 次週の方向性確認
- 改善提案の承認/却下

✅ **月次（1-2時間）:**
- 全体の進捗レビュー
- 戦略的な方向転換の検討
- システムの最適化

❌ **不要になること:**
- 毎日の細かい進捗管理
- 技術的な判断の大部分
- タスクの優先順位付け
- コードレビュー（自動化）
